# AI Medical Consultation Center (医宝医院)

基于大语言模型（LLM）的智能医疗咨询系统，为用户提供症状收集、诊断推理和治疗规划服务。

## 🚀 项目概述

医宝医院是一个基于多智能体架构的AI医疗咨询系统，通过自然语言处理技术为用户提供智能医疗咨询服务。系统采用流式输出、实时交互和补充信息请求机制，模拟专业医疗诊断流程。

## 🏥 核心功能

### 1. 智能症状收集
- 护士智能代理通过对话收集患者症状信息
- 结构化症状数据存储与处理
- 实时流式响应

### 2. 诊断推理引擎
- 双智能体角色扮演：推理专家 + 专科医生
- 基于症状的假设生成与验证
- 支持补充信息请求机制

### 3. 治疗规划系统
- 治疗规划师生成个性化治疗方案
- 合规性检查确保方案安全性
- 分阶段展示治疗建议

### 4. RAG知识检索
- 基于向量数据库的医学知识检索
- 支持PDF文档上传与知识库构建
- 为诊断推理提供参考信息

## 🛠️ 技术架构

### 后端架构
- **框架**: FastAPI + WebSocket
- **AI模型**: Baidu Qianfan (ERNIE系列) + DeepSeek
- **数据存储**: Qdrant向量数据库
- **核心模块**:
  - `main.py` - API入口点
  - `work.py` - 诊断工作流管理
  - `roleplay.py` - 双智能体对话管理
  - `patient_case.py` - 患者病例数据结构

### 前端架构
- **框架**: React + TypeScript + Vite
- **UI库**: Ant Design
- **通信**: WebSocket实时流式传输
- **核心组件**:
  - `ChatPage.tsx` - 主聊天界面
  - `websocket.ts` - WebSocket服务
  - `api.ts` - API客户端

## 📋 系统流程

```
用户输入 → 护士代理收集症状 → 诊断推理(双智能体) → 治疗规划(双智能体) → 最终报告
```

### 详细流程
1. **症状收集阶段**: 护士代理通过对话收集患者症状
2. **诊断推理阶段**: 
   - 推理专家分析症状，提出假设
   - 专科医生验证假设，可能请求补充信息
3. **治疗规划阶段**:
   - 治疗规划师制定治疗方案
   - 合规检测确保方案符合医疗规范
4. **结果呈现**: 向用户展示完整的诊断结果和治疗计划

## 🔧 开发环境设置

### 后端环境
```bash
# 安装依赖
pip install -r requirements.txt

# 启动服务
python main.py
# 或
uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

### 前端环境
```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 环境变量
```bash
# .env文件配置
ERNIE_API_KEY=your_qianfan_api_key
ERNIE_SECRET_KEY=your_qianfan_secret_key
```

## 📊 核心特性

### 1. 流式输出
- 实时显示AI响应内容
- 支持中断和继续

### 4. 知识检索集成
- 支持医学文献检索
- PDF文档上传与处理
- 为诊断提供参考依据
